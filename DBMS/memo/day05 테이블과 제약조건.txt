번호(PK)	이름	나이	아이디(UK)	구매번호(PK)       번호(FK)	날짜	상품

이러한 구조를 가지는 것을 Table, relation(오라클),Class라 부른다.

column(열,속성,필드)
	공통된 값들의 주제

Row(행,레코드,튜플)
	하나의 정보

PK(Primary Key)
	고유한 값(각 행의 구분점으로 사용된다.)
	중복이 없고 NULL값을 허용하지 않는다.
	* NULL : 아직 어떤 값을 넣을 지 모르겠다 라는 뜻을 가진 값

FK(Foreign Key)
	다른 테이블의 PK를 의미
	보통 테이블끼리 관계를 맺을 때 사용된다.
	중복이 가능하다 하지만 Table A에 없는 값을 Table B의 FK 컬럼에 사용할 수 없다.
	(pk에 5가 없는데 fk에서 5가 들어가면 문제)

Unique Key
	Null은 허용하지만 중복을 허용하지 않는 Key
